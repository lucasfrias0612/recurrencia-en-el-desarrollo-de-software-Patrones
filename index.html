Demostración por Inducción de 1+2+3+…​+n = n(n+1)/2

La inducción matemática demuestra que podemos subir tan alto como queramos en una escalera, si demostramos que podemos subir el primer peldaño (el "caso base") y que desde cada peldaño podemos subir al siguiente (el "paso" inductivo).

  • Caso base: se comprueba

  • 1·(1+1)/2 = 1·2/2 = 2/2 = 1

  • Cierto

  • Case general: se supone

  • Cierto

  • Caso inductivo:

  • ( (n+1)·( (n+1) + 1) ) / 2 =

  • ( (n+1)·(n+2) ) / 2 =

  • ( n^2 + n + 2·n + 2 ) / 2 =

  • ( n·(n+1) + 2·(n+1) ) / 2 =

  • n·(n+1)/2 + (n+1) =

  • 1+2+3+…​+n + (n+1)

  • Cierto

void quicksort(int datos[]){
  quicksort(datos, 0, datos.length);
}

void quicksort(int datos[],
    int izq, int der) {
  int pivote = datos[izq];
  int i = izq;
  int j = der;
  while(i < j) {
     while(datos[i] <= pivote && i < j)
      i++;
     while(datos[j] > pivote)
      j--;
     if (i < j) {
      int temp = datos[i];
      datos[i] = datos[j];
      datos[j] = temp;
     }
   }
   datos[izq] = datos[j];
   datos[j] = pivote;
   if (izq < j-1)
    quicksort(datos, izq, j-1);
   if (j+1 < der)
    quicksort(datos, j+1, der);
}

Modelo del Dominio

Definición Objetivos

  • Describe los conceptos más importantes (tipos de objetos) del contexto como objetos del dominio y los enlaces de éstos a otros

    • Objetos de negocio que representan cosas que son manipuladas en un negocio

    • Objetos del mundo real y conceptos que un sistema necesita hacer un seguimiento

    • Eventos que ocurrirán o han ocurrido

    • Nunca del software del sistema: vista, validador, …​!!!

  • Comprender la estructura y dinámica de la organización en la que el sistema se desarrollará (la organización destino)

  • Comprender los problemas actuales de la organización destino e identificar mejoras potenciales

  • Asegurar que clientes, usuarios finales y desarrolladores tienen una comprensión común de la organización destino

  • Obtener los requisitos del sistema para apoyar a la organización de destino

  • En talleres/reuniones por parte de analistas del dominio, expertos del dominio y personas con habilidad de modelar

Lenguaje Unificado de Modelado

  • UML: Lenguaje Unificado de Modelado (Unified Model Language) de la metodología

    • RUP: Proceso Unificado de Rational (Rational Unified Process)

    • Cada símbolo (léxico) relacionado (sintaxis) en un diagrama tiene un significado estándar (semática)

  • Para evitar ambigüedades

    • distintas personas interpretan lo mismo del mismo diagrama, opuesto a transparencias muy bonitas sin estandarizar, …​

    • la misma persona interpreta lo mismo del mismo diagrama pasado el tiempo, opuesto a una servilleta con un garabato sin estandarizar, …​
